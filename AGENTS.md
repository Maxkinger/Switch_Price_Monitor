# 项目执行要求

Codex 或其他自动化开发代理每次在本项目中执行任何代码、测试、配置或文档改动前，必须完整阅读本文件及 `docs/README.md`。

## 代码注释规范

- 所有新增或修改的源代码、测试代码、SQL 迁移和构建/运行配置必须包含中文的详细注释。
- 注释必须说明模块或函数的职责、关键数据约束、边界条件，以及安全或业务原因；不得只复述代码字面含义。
- 涉及认证、价格来源、汇率、通知、数据保留和数据库迁移的代码，必须特别说明其安全性或业务规则。
- 修改既有代码时，同时检查受影响的注释是否仍然准确；过期注释必须在同一次改动中修正。
- 所有代码审查和提交前验证都包含“注释与实现一致性”检查。

## 开发流程

- 功能改动遵循测试先行：先写失败测试，再实现最小代码，并运行相关测试。
- 不提交密钥、密码、恢复码、会话令牌或真实 Telegram 凭据。
- 任何架构或需求确认后，同步更新 `docs/` 内对应文档。
- 每次准备创建本地 Git 提交时，必须先向用户说明拟提交的变更范围并取得明确确认；获得确认后应在同一操作中完成本地提交与 `git push` 推送到远程仓库，不能只提交不推送。
