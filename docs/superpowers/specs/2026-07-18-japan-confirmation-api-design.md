# 日区订阅确认官方 API 复核设计规格

**状态：** 已部署并完成生产只读验收  
**日期：** 2026-07-18

## 1. 背景与问题

新增订阅向导可以通过任天堂日区公开软件搜索 API 找到 `Overcooked® 2 - オーバークック２ Nintendo Switch 2 Edition` 等日区候选，并在跨区核验中高置信度自动匹配。但是最终点击“确认订阅”时，现有服务会再次请求 `store-jp.nintendo.com` 的商品详情页，并要求页面返回可解析的公开结构化商品数据。

My Nintendo Store 可能向无浏览器的 Worker 返回 JavaScript 排队页或动态外壳，而非完整商品结构化数据。因此，候选虽然已经来自日区官方搜索 API，最终确认仍可能错误提示“商品链接不是该区任天堂官方链接，或公开商品信息无法验证”。这不是用户输入的链接一定错误，也不是跨语言匹配本身失败。

## 2. 目标与非目标

### 2.1 目标

- 日区候选在最终确认阶段不再依赖 My Nintendo Store HTML 的 JSON-LD 解析。
- 只使用任天堂官方公开接口重新证明日区商品的官方身份、商品类型、发行商、标题 ID、在售状态和日元币种。
- 日区英文/日文标题的自动匹配在最终确认时复用已部署的高置信度规则，避免因全文标题不同而误拒绝。
- 失败提示明确指出日区需要重新核验或稍后再试，不回显任天堂原始响应、网络细节或内部解析规则。
- 任一地区验证失败仍保持批量订阅零写入的原子性。

### 2.2 非目标

- 不改变 US、MX、BR、HK 的现有官方商品页复核方式。
- 不接入第三方价格站、翻译服务、AI、Nintendo Account、浏览器 Cookie 或额外管理员配置。
- 不为此次适配增加 D1 临时表、缓存或新的 Worker Secret。
- 不放宽手动选择和手动链接的既有安全边界。

## 3. 采用方案：日区双官方接口复核

### 3.1 最终确认数据流

```text
浏览器提交的日区官方 URL
        │
        ▼
Worker 提取并白名单验证 D<数字标题 ID>
        │
        ├── 日区公开软件搜索 API
        │     └── 使用已验证的搜索词重新取得同一标题 ID 的官方标题、发行商、类型
        │
        └── 日区公开价格 API
              └── 确认该标题 ID 为 JP、onsale、JPY
        │
        ▼
以两项官方响应构造最终持久化候选
        │
        ▼
按自动/人工来源执行跨区身份规则，再一次性写入 D1
```

Worker 从日区 URL 只提取精确 `https://store-jp.nintendo.com/item/software/D<数字>/` 格式中的数字标题 ID。该 ID 必须同时满足：

1. 日区公开软件搜索 API 当前返回同一下载版记录；
2. 日区公开价格 API 返回相同 `title_id`、`country: JP`、`sales_status: onsale` 和 `JPY` 整数价格。

因此，浏览器提交的标题、发行商、商品类型、封面和价格只用于表单形态校验，最终持久化值一律采用 Worker 当次重新获取的官方数据。

### 3.2 搜索复核的查询与精确匹配

日区搜索 API 当前没有已验证的“按标题 ID 直接读取”接口，因此对于跨区日区候选，Worker 使用已由默认区官方复核的锚点标题作为受控查询词，固定请求下载软件搜索；默认区本身为日区时，才使用其自身的官方标题。响应内必须要求：

- `id` 与 `nsuid` 都是相同的纯数字；
- `sform` 是下载版标识；
- 由该数字生成的官方 My Nintendo Store URL 与待确认 URL 完全一致；
- 标题、发行商和受控商品类型字段可完整解析。

若日区候选属于 `automatic`，Worker 会在当次搜索结果中重新计算严格或高置信度本地化身份，并要求待确认 URL 是唯一可自动匹配项。不能只因为浏览器提交了 `matchSource: "automatic"` 就相信它曾经是唯一候选。人工来源不会被这条自动匹配规则放宽。

搜索结果未包含该 ID、搜索接口不可用或响应结构变化时，不把浏览器字段当作替代证据。确认失败并要求管理员重新执行“核验其他地区”。

### 3.3 跨语言身份规则

最终确认中，`automatic` 的日区候选先经过上述双接口复核，再应用以下规则：

- 标题、商品类型、发行商完全规范化相同，仍按既有严格匹配通过；或
- 仅在商品类型一致、双方发行商都存在且相同、拉丁主标题标记相同、受控版本标记相同，并且该候选已由跨区发现阶段唯一自动确认时，接受本地化身份。

`manual_selection` 与 `manual_link` 不会从本次自动规则获益：它们仍须是经过本区官方复核的同类型商品，管理员必须明确选择或粘贴链接。升级包、DLC、季票、组合包和本体绝不因为名称相似而混配。

### 3.4 安全与失败边界

- 日区 API 请求只由 Worker 发起，使用固定任天堂官方端点、固定地区/语言参数和现有超时处理；浏览器不能指定端点、标题 ID、地区或货币。
- 商品价格 API 不包含标题、发行商和商品类型；因此不能单独作为身份验证依据，必须与官方搜索 API 配对使用。
- 确认前仍会重新读取当前启用地区；过期页面、篡改请求或设置变更均不能跳过地区覆盖校验。
- 默认区本身是日区时同样采用此适配器；其他地区仍采用现有详情页解析器。
- 异常文案仅使用服务端预设文本，例如“日区官方商品确认暂时失败，请重新核验其他地区后再试。”；不得把 HTTP 状态、原始 JSON、URL 查询或堆栈返回浏览器。

## 4. 接口与用户体验

`POST /api/products/confirm-subscriptions` 的请求和成功响应格式保持不变。对于日区确认失败，接口仍返回受控 `422 VALIDATION_ERROR`，但消息明确区分：

- 官方搜索记录无法再次确认：提示重新核验其他地区；
- 官方价格 API 暂不可确认：提示稍后重新核验；
- 搜索记录与 URL、ID、币种或在售状态冲突：提示该日区商品当前无法确认。

前端维持现有全局网络加载动画与错误提示位置；用户无需手动寻找日区 URL 或更改默认搜索区。

## 5. 验收与回归

- 日区官方搜索候选的 URL、下载版 ID、标题、发行商和类型会在确认前重新从 API 获取。
- 日区价格 API 的非 JP、非 JPY、非 onsale、错 ID 或缺价格响应均会拒绝，且 D1 不写入任何订阅数据。
- 美区英文与日区日文的 Overcooked 2 Switch 2 Edition 自动候选可在最终确认阶段通过高置信度身份规则。
- 缺失发行商、不同商品类型、不同主标题/版本标记或人工来源不得因本地化规则被自动放宽。
- US、MX、BR、HK 的商品页复核回归测试保持通过。
- 生产验证先只读完成搜索与跨区核验；真正点击“确认订阅”属于管理员明确授权的写入操作。
