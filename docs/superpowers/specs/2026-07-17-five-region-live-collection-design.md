# 五区真实价格采集设计规格

## 1. 目标与范围

本阶段将美国（US）、日本（JP）、墨西哥（MX）、巴西（BR）和香港（HK）的已确认地区商品接入真实采集链路。系统必须官方优先：每个地区独立尝试经过验证的任天堂公开入口；官方失败、超时、身份校验不通过或不存在可靠入口时，才按管理员在设置中启用并排序的第三方站点回退。

本阶段不允许根据标题猜测价格 ID，也不允许复用另一地区的价格 ID。所有价格采集都只在 Worker 服务端执行，浏览器仅调用本系统的受保护接口。

## 2. 地区适配器注册表

### 2.1 适配器职责

新增地区适配器注册表，为每个地区建立独立的官方采集策略，而非将单一区域的 URL 或 API 规则推广到五区。适配器可以使用下列任一已经过实测与 ADR 记录的任天堂公开入口：

- 返回价格、地区和货币的官方价格 API；
- 任天堂官方商品页的结构化公开数据；
- 能稳定读取商品身份与价格的任天堂官方公开页面。

日区继续使用已验证的公开价格 API，并仅接受该地区确认的 `officialPriceId`。美、墨、巴、港区分别先进行受控验证；验证证据、请求形状、身份字段和失败边界必须更新到 ADR-002 后，才能作为生产默认官方来源。

### 2.2 统一结果校验

适配器结果进入提供方链前必须包含来源、当前价、原价（如存在）、币种、地区和可用的商品身份字段。链路统一校验：

1. 返回地区与地区商品的 `regionCode` 相同；
2. 返回货币与该地区配置一致；
3. 标题、发行商和商品类型不与添加阶段已确认商品冲突；
4. 对支持价格 ID 的官方入口，返回 ID 必须与本区 `officialPriceId` 相同。

任一条件不满足即拒绝该结果，并继续下一个已启用来源。拒绝结果不得写入价格快照，也不得显示为官方价格。

### 2.3 第三方回退

官方来源失败后，提供方链按管理员设置的顺序尝试已启用第三方站点。第三方成功结果必须保存实际站点来源；页面、历史和 Telegram 日报须以“第三方：站点名”标记。第三方结果可用于价格展示、人民币换算和日报，但不得单独触发官方降价即时提醒。

所有来源失败时，系统保留最后一次成功快照，新增安全的采集失败日志，并让读取模型显示“数据可能过期”和最后成功时间。

## 3. 采集执行与调度

定时六小时采集和通过十五分钟冷却后的手动刷新必须复用同一个执行器。执行器依序读取启用订阅与地区商品、获取当天汇率、组装本区官方和第三方提供方链、写入成功快照或记录失败，并更新手动刷新请求状态。

每日中间汇率只读取一次；当日读取失败时复用最近一次成功汇率并标记汇率过期，不中断本次价格采集。成功快照为不可变记录，保留原始货币金额、人民币分值、汇率、原价与促销价、来源、采集时间和过期状态。

执行器在成功官方快照后比较上一条官方成功快照：只有价格下降时才创建即时降价事件。连续三次所有来源失败时创建一次异常事件；下一次成功时创建一次恢复事件。日报按既有时区与时间设置汇总当前价格与历史最低价。

## 4. 设置与可见状态

设置页保存第三方站点启用状态和优先级；任天堂官方来源始终排在第三方之前。管理员可见各地区官方适配器状态：官方可用、验证中，或官方不可用且正在使用第三方回退。状态不得泄露内部请求、Cookie、密钥或完整外部响应。

Telegram 配置、测试消息、时区、日报时间、税务州和历史留存策略继续沿用既有受保护设置边界。Token、Chat ID 与任何外部凭据均不得返回前端或写入日志。

## 5. 验收与测试

### 5.1 单元测试

每个地区适配器必须覆盖：正常价格、促销价与原价、地区或币种错配、商品身份错配、价格 ID 错配（如适用）、网络超时和无公开价格。提供方链还必须证明官方失败后按设置顺序回退，且第三方来源保持可见标记。

### 5.2 D1 与 Worker 集成测试

测试真实采集执行器对五区地区商品的安全枚举、不可变快照写入、汇率过期回退、手动刷新原子认领、连续失败与恢复事件去重。定时处理器必须验证六小时采集、手动刷新消费、日报调度和待发送通知投递互不重复。

### 5.3 受控环境验收

在测试环境选取同一商品的五区官方链接或确认商品数据，逐区确认以下任一明确结果：官方价格已验证；官方失败后使用指定第三方站点并标记；或所有来源失败且保留上次价格。随后验证历史快照、手动刷新冷却、Telegram 测试消息、日报历史最低价和异常/恢复通知。

## 6. 约束与安全边界

- 不使用浏览器端抓取、无头浏览器或绕过任天堂访问控制的方式。
- 每次外部请求设定超时，网络类临时故障只重试一次。
- 适配器准入和站点结构变更均需先更新 ADR-002 的验证证据，再进入生产默认链路。
- 所有新增或修改的代码、测试、SQL 和配置遵循项目中文详细注释与测试先行要求。
