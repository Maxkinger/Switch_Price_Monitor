# 跨语言地区商品高置信度识别设计规格

**状态：** 已实施，待部署与生产只读验收
**日期：** 2026-07-18

## 1. 背景与目标

新增订阅的跨区核验以默认区候选为锚点。现有自动规则只比较小写化后的完整标题、商品类型和发行商；因此美区 `Overcooked! 2 – Nintendo Switch 2 Edition` 与日区 `Overcooked® 2 - オーバークック２ Nintendo Switch 2 Edition` 会因日文别名和 `®` 等排版差异无法自动对应，虽然它们同为 Team17 的同类型 Switch 2 Edition 商品。

本次目标是在不降低自动订阅安全边界的前提下，识别这类具有充分共同官方身份信号的本地化商品，并在不能唯一确认时改善人工候选的可读性。

## 2. 方案选择

评估过三种方案：

1. 继续只做完整标题严格匹配：实现最简单，但会持续把大量真实的本地化商品交给人工选择。
2. 使用外部翻译或 AI 语义匹配：覆盖面较大，但会引入非官方数据、成本、不可重复的判断和错误自动订阅风险。
3. **受控的高置信度跨语言身份信号（采用）：** 仅使用任天堂官方候选字段，组合规范化拉丁主标题、受控版本标记、商品类型、发行商和唯一性判断；无法唯一确认时仍由管理员选择。

## 3. 匹配规则

### 3.1 严格匹配保持不变

完整规范化标题、商品类型以及双方都有时的发行商相同，仍是最高优先级的自动匹配方式。既有结果和审计来源不变。

### 3.2 高置信度本地化匹配

当严格匹配失败时，系统仅在以下全部条件成立时自动确认：

- 地区搜索结果已通过本区任天堂官方 URL 白名单与商品类型过滤。
- 锚点和候选的商品类型相同。
- 双方均有发行商时，发行商经 Unicode NFKC、大小写与空白规范化后相同；任一方缺少发行商时不得因本规则自动确认。
- 两个标题都可提取相同的、至少含三个拉丁字母和一个数字的主标题标记，例如 `Overcooked 2`；仅有 `Nintendo Switch 2 Edition` 等泛版本词不足以成立。
- 两个标题都包含相同的受控版本标记，例如 `Nintendo Switch 2 Edition`；该标记只作为附加证据，不能单独识别商品。
- 全部地区候选中恰好一项满足上述条件。

标题提取只用于瞬时匹配，不修改页面展示标题、最终保存的官方标题或地区商品 URL。日文别名如 `オーバークック２` 不要求被翻译；共同存在的拉丁主标题和官方元数据已提供可审计的身份依据。

### 3.3 人工核验的候选呈现

若高置信度规则没有得到唯一结果，系统仍返回同类型的官方候选列表。列表按如下分层排序：

1. 高置信度信号命中但不唯一的候选；
2. 发行商相同且与锚点共享拉丁主标题词的候选；
3. 其余同类型官方候选，按既有标题与 URL 的稳定顺序排列。

页面默认显示第一层和第二层候选；若仍没有高相关候选，显示固定数量的前置候选。其余结果位于“更多官方候选”折叠区。折叠只改变展示，不改变 Worker 返回的候选、最终二次解析或管理员选择权限。

## 4. 安全与失败边界

- 不调用第三方搜索、翻译、AI、Nintendo Account 或浏览器 Cookie。
- 任何发行商、类型、官方地区 URL 或唯一性冲突都会拒绝自动确认，回退到人工选择。
- 升级包、季票、DLC、组合包与本体即使拥有相同游戏名称也不得互相自动匹配。
- 手动选择、手动官方链接、显式跳过和最终确认时的商品页二次解析保持原有行为。

## 5. 测试与验收

- 服务测试覆盖上述 Overcooked 美日标题在唯一、同发行商、同类型、同版本标记下得到 `automatic`。
- 覆盖缺失发行商、不同商品类型、仅共享版本标记、多个高置信度候选均不得自动确认。
- 覆盖人工列表的分层稳定排序与前端“更多官方候选”展开行为。
- 回归现有严格匹配、官方 URL 白名单、人工选择和最终确认服务测试。
- 生产验收只读取候选和核验结果；除非管理员明确点击最终确认，不创建订阅。

## 6. 结果

同一商品的英文与日文官方标题可以在有足够独立身份信号时自动对应；证据不足时，系统仍把正确候选置于最容易核验的位置，并保留管理员的最终决定权。
