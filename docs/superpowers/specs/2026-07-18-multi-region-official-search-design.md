# 多地区任天堂官方搜索与自动监控设计规格

**状态：** 实施中；五区官方搜索与跨区发现分流已完成并通过固定夹具回归，最终人工确认校验与界面待实施
**日期：** 2026-07-18

## 1. 目标与问题

新增订阅目前只实际调用美区的任天堂官方搜索。虽然向导会按保存的 `settings.enabledRegions` 请求其他地区，但日区、墨西哥区、巴西区和香港区被适配器直接标记为不可用，管理员只能手动粘贴链接。

本设计要求：管理员在美区选择一个官方商品后，系统主动查询所有启用地区的任天堂官方商品；对唯一且可安全确认的结果直接加入该订阅的监控地区。手动输入官方链接只在没有可验证结果时保留为最后兜底。

## 实施记录

- 2026-07-18：已将 US、MX、BR 的公开任天堂 Algolia 游戏索引收敛为 Worker 内不可变地区档案；档案分别绑定 `store_game_en_us`/`USD`/`/us/`、`store_game_es_mx`/`MXN`/`/es-mx/`、`store_game_pt_br`/`BRL`/`/pt-br/`。固定响应测试覆盖对应索引、币种及 URL 白名单。
- 2026-07-18：已验证并实现 HK 的 `https://www.nintendo.com/hk/search?k=…` RSC `software.items` 读取。候选只允许其官方 `ec.nintendo.com/HK/zh/titles/{NSUID}` 模板和数字 NSUID；eShop 页面以公开 `search.*` 元数据二次确认，缺少可验证价格时保留空值。
- 2026-07-18：已验证并实现 JP 的 `https://search.nintendo.jp/nintendo_soft/search.json` 公开软件 API。候选只接受 `*_DL` 下载版和相等的数字 `id/nsuid`，再按官方映射生成 `store-jp.nintendo.com/item/software/D{id}/`；实体版或聚合 ID 不生成商品链接。
- 2026-07-18：发现服务已区分唯一严格自动匹配与人工选择：同地区、同类型且符合官方商品 URL 白名单的候选会以标题及 URL 稳定排序。只有唯一标题/类型/发行商严格匹配可自动加入；本地化标题、多个严格匹配等歧义保留为管理员候选卡；搜索不可用或已验证空集合才显示官方链接兜底。

## 2. 范围与非目标

### 2.1 本次范围

- US、MX、BR、HK、JP 五区均有独立的任天堂官方搜索适配器或已验证官方搜索请求配置。
- 美区仍是默认搜索区；启用地区仍只由 `settings.enabledRegions` 决定，浏览器不能扩大范围。
- 每个已选商品会向其他启用地区自动查询；唯一严格匹配的结果直接成为 `automatic` 确认映射。
- 有官方候选但不能唯一自动匹配时，显示该区候选列表供管理员选择；管理员选择后以 `manual_selection` 记录。
- 没有候选、请求失败或官方结构无法验证时，显示粘贴本区任天堂官方链接与显式跳过入口。
- 所有候选和链接在最终确认前继续经过 Worker 的地区主机、路径、商品类型与价格 ID 校验；订阅仍以单个 D1 批次原子创建。

### 2.2 非目标

- 不使用第三方搜索、价格网站、Nintendo Account、Cookie、购买记录或浏览器自动化。
- 不把静态 HTML 中的 “Loading” 当作搜索结果，也不在浏览器直接请求任天堂。
- 不自动接受标题、类型或发行商存在冲突的跨区商品；这种情形只能由管理员从候选列表明确选择，或使用链接/跳过。
- 不修改既有订阅、价格快照、目标价或通知状态；已有订阅的补全入口沿用同一解析规则。

## 3. 官方检索架构

### 3.1 地区搜索配置

Worker 新增只读的 `OfficialRegionalSearchProfile` 配置。每项包含地区代码、任天堂官方请求地址、公开应用配置、索引/语言参数、货币、官方商品 URL 白名单与受控 JSON 解析器。配置只能在服务端定义，绝不由浏览器请求体提供。

- US、MX、BR 的公开搜索页面采用同一任天堂动态搜索应用；它们使用各自地区、语言与货币配置，不能把 US 索引结果复用为 MX 或 BR 候选。
- HK 使用 `https://www.nintendo.com/hk/search?k=…` 的 RSC `software.items`；它仅接受 `hongkong` 记录和官网实际返回的 `ec.nintendo.com/HK/zh/titles/{NSUID}` 模板，随后用该 eShop 页公开元数据二次验证。
- JP 使用 `https://search.nintendo.jp/nintendo_soft/search.json` 的公开软件索引；只有数字下载版 ID 才能映射到 `store-jp.nintendo.com` 商品 URL。

适配器遇到网络超时、非成功 HTTP、字段结构变化、货币不符、非官方链接或未知商品类型时，只返回该区的受控不可用状态，不回显外部正文。

### 3.2 候选身份与确认规则

每项候选必须包含：地区、官方 HTTPS 商品 URL、标题、受控商品类型、地区币种、当前价/常规价（可空）、发行商（可空）和封面（可空）。

自动确认使用严格匹配：规范化标题和商品类型相同；双方都有发行商时发行商也必须相同。仅一项严格匹配时，系统自动选中并将该地区带入最终订阅载荷。

当结果有多个严格匹配、只有语言化标题差异，或返回若干同类官方候选时，页面展示候选列表而不猜测。管理员显式点选某项后，Worker 重新解析其官方 URL；`manual_selection` 允许标题随地区语言不同，但仍要求商品类型相同、地区和商品 URL 合法，且将管理员选择作为审计来源。发行商差异显示为候选信息而不作为手动选择的阻断条件。

当搜索没有候选或无法验证时，页面只展示本区官方链接输入框和“跳过此区”。手动链接同样必须通过 Worker 解析，并与默认区商品类型一致；跳过必须由管理员显式选择。

### 3.3 新增订阅数据流

```text
默认区官方搜索 → 管理员选择商品
                         │
                         ▼
      Worker 按 settings.enabledRegions 查询各区官方搜索适配器
                         │
           ┌─────────────┼──────────────┐
           ▼             ▼              ▼
       唯一严格匹配   多项/语言差异      无结果/不可用
           │             │              │
           ▼             ▼              ▼
        自动加入      候选列表选择     官方链接或跳过
           └─────────────┴──────────────┘
                         │
                         ▼
             Worker 二次验证并原子创建订阅
```

最终确认按钮只在所有启用地区都已自动确认、人工确认或显式跳过时可用。浏览器只提交候选或跳过代码，不提交地区范围、商品 ID 或价格 ID。

## 4. 既有订阅与界面

“补全已启用地区”复用同一地区搜索服务。已保存的官方地区商品继续作为锚点；唯一严格匹配自动加入补全草稿，候选不唯一时显示可选择列表。最终“确认补全”仍要求管理员点击，并仅追加缺失地区，不替换既有数据。

在新增向导中，自动加入的地区应标注“已自动加入监控”；候选列表保持现有卡片的封面、标题、类型、发行商与原价/折扣价布局。无结果的手动链接字段不应在存在候选时抢占主要操作。

## 5. 测试与验收

- 为每区搜索适配器使用固定官方响应夹具，测试请求配置、地区 URL 白名单、币种和字段解析；测试不得访问真实任天堂网络。
- 覆盖唯一严格匹配自动加入、多个候选须选择、语言化标题允许人工确认、错误类型/非官方 URL 被拒绝、无结果显示官方链接/跳过。
- 覆盖确认服务：自动映射仍使用严格身份校验；人工选择和链接必须是本区官方候选且类型一致；任一无效地区使 D1 原子写入整体失败。
- 生产验收使用一个已登录的测试订阅：验证 MX、BR、HK、JP 的候选或安全兜底显示；只有管理员主动点击最终确认时才验证写入。不得记录 Cookie、密码、恢复码或 Telegram 凭据。

## 6. 结果

完成后，“我选择的监控地区”会成为新增订阅的自动查询范围：可安全识别的五区商品无需复制链接即可直接被监控；只有任天堂官方数据无法确认时，管理员才需要手动选择、提供官方链接或跳过。
