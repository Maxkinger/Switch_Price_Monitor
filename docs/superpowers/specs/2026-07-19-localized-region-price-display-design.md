# 仪表盘与详情页地区中文名及官方价格格式设计规格

**日期：** 2026-07-19
**状态：** 已实现，本地草图已由管理员确认
**范围：** 仪表盘与订阅详情页的地区名称、价格文字和本地草图；不改变价格采集、数据库、API 金额或订阅数据。

## 1. 目标

管理员阅读仪表盘和订阅详情时，应看到中文地区名称和与该地区任天堂商店约定一致的本地货币文字，而不是 `US · USD` 这类代码组合。价格信息只出现一次；地区名称用于区分美国区与墨西哥区同为 `$` 的情况。

## 2. 固定展示规则

| 地区代码 | 地区名称 | 价格示例 |
| --- | --- | --- |
| `US` | 美国区 | `$ 39.99` |
| `MX` | 墨西哥区 | `$ 39.99` |
| `JP` | 日本区 | `1,999 円（税込）` |
| `BR` | 巴西区 | `R$ 99.00` |
| `HK` | 香港区 | `HKD 198` |

货币符号或代码与金额之间一律保留一个空格。日本价格的税收说明属于价格文字的一部分，不另加货币代码。香港价格按已确认的官方文字使用 `HKD` 加整数；如果未来官方 API 返回非整港元金额，保留最多两位小数而不截断金额。

未知地区不得猜测中文名称或货币符号：地区名继续显示原地区代码，价格继续使用既有的安全后备格式。

## 3. 页面和组件边界

1. 在既有前端展示模型中集中提供地区代码到中文名的映射与本地价格格式化函数。
2. 仪表盘卡片将地区简称替换为中文名称；价格继续只显示一次。
3. 订阅详情页的地区价格标题从 `US · USD` 等组合替换为中文地区名称；价格行使用同一格式化函数。
4. 订阅向导候选卡不在本次范围内，避免把候选搜索结果的官方原始价格文案与已订阅地区价格的展示规则混在一起。
5. 草图使用静态的《Overcooked! 2 – Nintendo Switch 2 Edition》五区样例，只用于本地可视化验证，不创建订阅、快照或官方请求。

## 4. 数据与安全边界

- Worker 仍传输最小货币单位、币种和来源；客户端只格式化已验证的展示数据，不重算汇率、不请求任天堂、不猜测价格。
- 不新增 D1 表、迁移、Cron、价格来源或公开接口字段。
- 价格来源标记、采集时间、历史最低价、过期状态和删除逻辑保持不变。

## 5. 测试与验收

- 纯函数测试覆盖五个地区的中文名称、金额格式、符号/代码与金额之间的一个空格，以及未知地区回退。
- 仪表盘 DOM 测试验证中文地区名和 `$ 39.99`；订阅详情 DOM 测试验证不再显示 `US · USD` 并展示日本与香港格式。
- 本地草图以桌面表格显示五区当前价格、人民币估算和地区历史最低价；语义 DOM 与控制台检查通过，视觉呈现已由管理员确认。
- 完整 Worker/DOM 回归、TypeScript、生产构建和 `git diff --check` 必须通过后才能提交。

## 6. 明确不做的事

- 不改变历史快照金额、地区代码、官方价格校验或汇率计算。
- 不为美国和墨西哥强行添加不同 `$` 前缀；中文地区名已经提供地区语义。
- 不在此变更中部署生产、修改版本号或改变 Telegram 文案。

## 7. 实施记录

- `dashboard-view-model.ts` 已集中提供地区中文名称与地区级官方价格格式；地区代码与币种不匹配时保留既有安全回退，避免把异常采集数据伪装成官方价格。
- 仪表盘与订阅详情的地区价格、历史最低价和跨区历史最低价均使用同一格式化函数；订阅管理表单继续保留机器地区代码，不改变管理接口。
- 《Overcooked! 2 – Nintendo Switch 2 Edition》五区静态草图只使用测试夹具金额，不会创建订阅、价格快照或官方请求。
