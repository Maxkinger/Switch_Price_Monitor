# 日区升级包官方关系发现设计规格

**状态：** Browser Run 三次隔离复验通过；允许进入生产集成设计，人工官方链接兜底继续保留
**日期：** 2026-07-18

## 1. 背景与已验证事实

日区任天堂公开软件搜索 API 会返回 `Overcooked® 2 - オーバークック２ Nintendo Switch 2 Edition`，并以 `upgrade: 1` 表示该完整版存在升级路径，但不会返回升级通行证自己的商品 ID。用“アップグレードパス”或升级包名称检索时，搜索 API 仍不会把该升级包作为独立候选返回。

任天堂日区公开详情页 `https://store-jp.nintendo.com/item/software/D70010000106252/` 在完整浏览器中展示“アップグレードパス”入口，目标为升级包官方地址 `https://store-jp.nintendo.com/item/software/D70050000064985/`。日区官方价格 API 可分别验证完整版 ID `70010000106252` 的 JPY 价格和升级包 ID `70050000064985` 的 JPY 价格，但价格响应不包含两个商品之间的关系。

普通 Cloudflare Worker `fetch` 请求该详情页时只得到 JavaScript 排队页，响应中没有可验证的升级链接。因此，先前“普通 Worker 直接解析官方详情 HTML”的假设不成立，不能据此进入生产实现。Cloudflare Browser Run 能运行完整浏览器，但官方说明这类浏览器仍会被识别为自动化访问，不能假设它一定能通过任天堂排队页或反机器人策略，必须先做隔离、只读的可行性验证。

## 2. 当前阶段目标与非目标

### 目标

1. 使用独立的 Cloudflare Browser Run 探针，验证完整浏览器能否在 30 秒内从指定日区官方详情页读取明确的“アップグレードパス”链接。
2. 对同一正向样本连续执行三次独立验证，只有三次均返回同一升级包 URL `D70050000064985` 才判定技术路径可行。
3. 把验证结果作为后续生产架构设计的输入；验证本身不得改变生产 Worker、D1、Cron、订阅或价格历史。

### 非目标

- 本阶段不把 Browser Run 接入商品搜索、跨区核验、最终订阅确认或六小时价格采集。
- 不部署到现有生产地址 `switch-price-monitor.cchccp.workers.dev`，不修改生产 Worker 的绑定和依赖。
- 不登录 Nintendo Account，不使用用户浏览器会话，不执行购买、加入购物车或账号操作。
- 不保存 Cookie、localStorage、IndexedDB、排队令牌、完整 HTML、响应正文、截图或网络归档。
- 不使用第三方索引、搜索引擎缓存或静态升级包 ID 对照表补齐验证结果。
- 不把一次成功、人工观察到链接或已知升级包 ID 视为生产能力证明。

## 3. 方案比较与决策

### 方案 A：Cloudflare Browser Run 隔离探针（当前采用）

使用 Cloudflare 官方 `@cloudflare/playwright` 和 Browser Binding 启动无持久状态的浏览器，只访问已确认的任天堂日区公开商品页，等待目标关系链接出现并执行严格白名单提取。它与既有 Cloudflare 架构一致，能够验证 JavaScript 渲染和排队页后的真实可达性，但仍可能被任天堂识别为机器人，因此结果必须以三次真实运行决定，而不是凭能力说明推断。

### 方案 B：管理员手工粘贴日区升级包官方链接

沿用现有人工官方链接兜底，不增加浏览器成本，可靠性边界最清楚；缺点是无法达到“只输入游戏名称并自动补齐启用地区”的目标。若方案 A 验证失败，本方案仍是安全默认行为。

### 方案 C：第三方站点发现升级包 ID

第三方索引可能直接提供日区升级包 ID，但项目已确认商品发现只使用任天堂官方来源，且第三方站点尚未通过 API、许可与条款准入，因此本阶段不采用。

普通 Worker 静态 HTML 解析不再作为候选方案：已验证响应是排队页，缺少关系证据。

## 4. 隔离验证架构

### 4.1 运行边界

探针应位于独立临时目录，使用独立 Worker 名称和 Browser Binding，不引用生产 D1、Static Assets、Cron、Secrets 或生产域名。优先使用 `wrangler dev --remote` 的临时开发会话；若当前 Wrangler 或账号环境不能在远程开发会话中提供 Browser Binding，应立即停止，并在任何一次性 Worker 部署前单独取得管理员授权。

探针配置采用 Cloudflare 当前官方要求：`@cloudflare/playwright`、`nodejs_compat`、不早于 `2025-09-15` 的兼容日期和专用 Browser Binding。当前官方免费额度为每天 10 分钟浏览器时间、最多 3 个并发浏览器，免费账户新建浏览器实例的频率限制为每 20 秒 1 个；三次验证须串行执行并遵守该限制，不并发启动。

### 4.2 单次探测流程

每次探测均创建全新的浏览器与隔离上下文，不读取或复用前一次的 Cookie、缓存和存储：

1. 只导航到 `https://store-jp.nintendo.com/item/software/D70010000106252/`。
2. 最多等待 30 秒，允许页面自身完成普通 JavaScript 渲染；不点击验证码、不模拟登录、不尝试规避排队或反机器人控制。
3. 只检查可见链接文本包含 `アップグレードパス` 的锚点。
4. 目标 URL 必须同时满足 HTTPS、精确主机 `store-jp.nintendo.com`、路径正则 `^/item/software/D[0-9]+/?$`；任天堂官方页面实际可能省略末尾斜杠，校验器接受零个或一个末尾斜杠后统一规范化为带斜杠 URL。查询参数和片段不参与商品身份，其他路径、主机、端口或凭据仍严格拒绝。
5. 去重后必须恰好得到一个合规链接；零个或多个都返回失败，不能按 DOM 顺序选择。
6. 无论成功、超时或异常都立即关闭页面、上下文和浏览器，不保留会话。

### 4.3 最小输出

单次探针只允许返回以下公开、非敏感字段：

- `status`：`success` 或归一化失败类型；
- `upgradeUrl`：成功时的完整官方升级包 URL；
- `linkText`：成功时页面公开显示的升级链接文字；
- `elapsedMs`：从开始导航到完成提取或失败的毫秒数。

日志不得包含页面 HTML、Cookie、存储状态、排队令牌、请求头、响应头、截图、堆栈或任天堂响应正文。错误只归一化为 `timeout`、`blocked-or-missing`、`multiple-matches`、`invalid-official-url`、`browser-launch-failed` 或 `unexpected-error`。

## 5. 成功门槛与后续分支

必须连续完成三次独立浏览器探测，并同时满足：

1. 每次都在 30 秒内返回 `success`；
2. 每次都只发现一个合规升级链接；
3. 三次 `upgradeUrl` 完全一致；
4. 三次结果均为 `https://store-jp.nintendo.com/item/software/D70050000064985/`；
5. 探针过程没有使用持久会话、人工干预、验证码绕过或非官方数据源。

全部通过只表示“Browser Run 可以进入正式生产设计”，不等于生产功能已批准。下一阶段必须另写生产设计，至少评估调用频率、并发与费用、缓存和失败回退、最终确认的二次证明、监控告警，以及任天堂页面结构和自动化访问政策变化。

任意一次超时、链接缺失、多个匹配、URL 不一致、页面要求人工干预或结构校验失败，即判定本轮技术路径不可准入；停止生产集成，继续使用管理员手工粘贴官方链接。第三方 ID 发现仍须另行完成来源许可与需求确认，不能自动启用。

### 5.1 2026-07-19 实际验证结论

在修复 Wrangler/Playwright CDP 协议版本不兼容和官方 URL 末尾斜杠规范化后，单次正向验证成功。随后三次全新远程浏览器实例均在 30 秒内返回唯一规范 URL `https://store-jp.nintendo.com/item/software/D70050000064985/`，且相邻实例满足已批准的创建间隔；机械校验为 `count: 3`、`allSuccess: true`、`oneUrl: true`。

因此方案 A 已达到“允许进入正式生产设计”的门槛。验证仍只覆盖一个已知正向样本，不授权把临时探针复制进生产，也不删除人工官方链接兜底。生产集成必须另行设计和确认。

## 6. 测试与执行约束

探针实现仍遵循项目测试先行和中文详细注释规则：

- 先以纯函数测试覆盖官方 URL 白名单、可见文字约束、零/多结果拒绝和失败类型归一化，再实现最小提取逻辑。
- 以模拟页面测试证明普通 DLC、季票、推荐商品、外部主机、错误路径和隐藏链接不会成为结果。
- 本地类型检查和单元测试通过后，才允许启动隔离的远程浏览器验证。
- 三次真实运行须分别记录允许字段与开始时间，实例启动至少间隔 20 秒；不得为了通过门槛重用已完成排队的持久上下文。
- 可行性验证结束后终止远程开发会话。临时探针不合并到生产应用，不创建 D1 数据，不触发订阅确认或价格刷新。
