# 日区升级包官方关系发现设计规格

**状态：** 已确认，待实施计划与编码
**日期：** 2026-07-18

## 1. 背景与证据

日区任天堂公开软件搜索 API 会返回 `Overcooked® 2 - オーバークック２ Nintendo Switch 2 Edition`，并以 `upgrade: 1` 表示该完整版存在升级路径，但不会把升级通行证作为同一名称搜索中的独立候选返回。

任天堂日区完整版详情页 `https://store-jp.nintendo.com/item/software/D70010000106252/` 明确展示“アップグレードパス”入口，并链接到升级包自己的官方地址 `https://store-jp.nintendo.com/item/software/D70050000064985/`。因此，完整版 ID `70010000106252` 只能作为关系根；价格采集和订阅必须使用升级包自己的 ID `70050000064985`。

现有实现只把日区软件搜索记录转换为候选，最终确认还要求同一候选 URL 重新出现在软件搜索结果中。对于仅通过完整版详情公开的升级包，这两个条件均无法成立，导致系统回退手工链接或在最终确认阶段拒绝。

## 2. 目标与非目标

### 目标

1. 当跨区目标是 `upgrade-pack` 且日区直接搜索没有同类型候选时，从日区官方 Switch 2 Edition 详情的一层升级关系发现升级包。
2. 唯一、完整验证的升级包自动加入；多个已验证升级包显示候选供管理员选择。
3. 保存和采集始终使用升级包自己的日区 URL 与价格 ID，不把完整版价格当作升级包价格。
4. 最终确认重新验证同一官方关系和价格 ID，不能信任浏览器提交的标题、类型或 `automatic` 状态。

### 非目标

- 不假设所有升级包一定采用相同页面结构；无法验证时继续使用手工官方链接兜底。
- 不使用第三方索引、搜索引擎缓存、Nintendo Account、Cookie 或用户浏览器会话。
- 不引入浏览器渲染服务、VPS 抓取服务或静态升级包 ID 对照表。
- 不递归展开升级包、DLC 或组合商品的第二层关系。
- 不修改数据库结构、既有订阅或历史价格。

## 3. 方案比较与决策

### 方案 A：解析官方完整版的一层升级关系（采用）

先用日区公开软件搜索 API 找到带 `upgrade: 1` 的 Switch 2 Edition 下载版，再读取其官方详情页中明确标记为“アップグレードパス”的链接，最后用日区公开价格 API 验证链接 ID。该方案保持全链路任天堂官方来源，能支持多款遵循相同公开关系契约的游戏，也能在结构变化时安全失败。

### 方案 B：浏览器渲染或独立抓取服务

完整浏览器更容易处理 JavaScript 与排队页，但会新增服务器、认证、资源消耗、可用性与隐私边界。当前需求只需读取官方页面中已服务端输出的一层链接，不引入该架构。

### 方案 C：维护升级包 ID 对照表

硬编码已知 ID 实现最快，但无法自动支持多款游戏，映射过期后还可能采集错误商品。该方案不作为生产能力。

## 4. 架构与职责

### 4.1 日区升级根搜索

现有日区搜索适配器继续只返回浏览器所需的普通官方候选，不在共享 DTO 中增加内部关系字段。新增窄接口 `JapaneseUpgradeRootSearch`，复用同一固定日区公开软件搜索端点和下载版校验规则，但只在关系兜底中返回服务端内部的升级根证据：官方 URL、标题、发行商、商品形态以及经过严格整数校验的 `upgrade: 1`。

该接口最多为一次跨区核验额外执行一次基础系列查询，并最多返回五个 `BEE_DL` 或 `HAC_DL` 下载版根；实体版、`DL_DLC` 组合商品、未知形态及 `upgrade` 非整数 `1` 的记录均拒绝。升级能力证据不返回浏览器、不写入 D1；它只表示“允许尝试读取一层升级关系”，不能直接生成升级包候选或价格 ID。

### 4.2 日区升级关系解析器

新增独立的 `JapaneseUpgradeRelationResolver`，职责仅为从已验证的日区下载版详情读取一层升级包引用：

- 输入必须是日区、JPY、`game` 类型、官方 `store-jp.nintendo.com/item/software/D{数字}/` URL，并带有官方搜索证明的升级能力。
- 请求不携带 Cookie、账号或购买信息，只接受 HTTPS、精确官方主机与软件路径。
- 页面只扫描明确的升级通行证关系区域和可见名称包含 `アップグレードパス` 的链接；普通 DLC、季票、组合商品、推荐商品与任意数字文本均不能成为升级包。
- 目标链接必须再次通过精确官方 URL 校验，并提取纯数字 `7005...` 类价格 ID；ID 前缀只作为当前样本证据，生产校验仍以完整数字、官方路径和价格 API 回显为准，不用宽松前缀代替身份验证。
- 候选标题来自官方升级链接文本；发行商来自同一官方搜索根和明确的一层升级关系，不从默认区浏览器数据复制。
- 单个根最多接受五个去重升级引用，不递归读取目标页面的更多关系。
- 页面为 JavaScript 排队页、缺少关系区、链接结构不完整或超时时返回“不可完整证明”，不猜测 ID。

### 4.3 跨区发现服务

关系兜底只在以下条件全部满足时启动：

1. 默认区锚点类型为 `upgrade-pack`；
2. 日区直接搜索没有任何同类型候选；
3. 可从受控 `Nintendo Switch 2 Edition Upgrade Pack` / `アップグレードパス` 后缀得到基础系列查询；
4. 日区官方搜索返回不超过五个下载版根。

根候选必须与锚点满足相同拉丁系列标记、同发行商规范化值和 Switch 2 Edition 标记，并由官方字段明确表示可升级。关系解析后的升级包还必须满足 `upgrade-pack` 类型、相同系列标记与同发行商关系证据。

- 所有相关根均完整返回且恰好一个升级候选时，标记为 `automatic`。
- 得到多个完整验证候选时，按相关性稳定排序并显示候选列表，由管理员选择。
- 只有部分根可读取时，可显示已完整验证的候选供人工选择，但不能形成 `automatic`；没有完整候选时显示手工官方链接。
- 直接搜索已经返回同类型候选时不执行关系兜底，避免重复请求和两套证据互相覆盖。

### 4.4 日区最终确认

现有 `JapaneseSubscriptionConfirmationService` 对普通游戏和组合商品继续使用软件搜索 API + 价格 API 双重确认。对于关系发现的 `upgrade-pack`，改为：

1. 根据默认区官方锚点重新执行同一受控日区根搜索；
2. 重新读取一层升级关系；
3. 要求待保存的升级包官方 URL 在本次完整验证结果中精确存在；
4. `automatic` 来源还必须重新保持唯一；人工选择只放宽唯一性，不放宽官方关系、类型和 URL 校验；
5. 用升级包 URL 提取自己的 ID，并要求价格 API 返回同一 `title_id`、`country: JP`、`currency: JPY`、`sales_status: onsale` 和有效日元价格。

任一步失败都发生在 D1 写入前，并拒绝整个确认批次。浏览器无需提交关系根 URL，服务端可以从锚点和固定官方搜索重新建立证明，因此不增加数据库字段或 API 请求字段。

## 5. 错误处理与安全边界

- JavaScript 排队页、HTTP 非成功、超时和页面结构变化统一返回安全兜底，不向浏览器回显响应正文、队列令牌、内部 URL 参数或堆栈。
- `upgrade: 1` 本身不能生成候选、ID、价格或 automatic 状态。
- 完整版的 `700100...` ID 绝不能保存为升级包价格 ID；最终快照只能使用升级链接中的 ID。
- 多个关系不能按页面顺序擅自选择；重复 URL 去重后仍须遵守唯一性规则。
- 关系解析不访问非任天堂主机，不跟随页面提供的任意外链，不形成通用请求代理。
- 临时失败不写入半成品地区映射；管理员可重试或使用现有手工官方链接流程。

## 6. 测试与验收

测试先行，至少覆盖：

- 日区直接搜索已有升级包候选时不触发关系兜底。
- 唯一 `upgrade: 1` 完整版从官方详情发现正确 `D700500...` URL，并经 JP/JPY/onsale/同 ID 价格响应后成为升级包候选。
- `upgrade: 0`、缺失字段、实体版、未知形态和非官方根 URL 不展开。
- 同页普通 DLC、季票、组合商品和升级包并存时只接受明确的 `アップグレードパス`。
- 外部主机、错误路径、非数字 ID、错地区、错币种、错回显 ID 和非在售状态全部拒绝。
- 一个唯一完整候选自动加入；多个完整候选进入人工候选列表；部分根不可读时不得自动加入。
- 排队页、HTTP 失败、超时、缺少关系区和页面结构异常回退手工链接。
- 最终确认重新发现相同 URL；关系消失、候选不再唯一或价格验证失败时四张订阅业务表零写入。
- 回归日区普通游戏、美食家 `DL_DLC`、港区一层关系发现、US/MX/BR 搜索和价格预览。

生产只读验收使用美区 `Overcooked! 2 - Nintendo Switch 2 Edition Upgrade Pack` 作为锚点，要求日区自动匹配升级包自己的 URL `D70050000064985`，价格来源预览显示任天堂官方 JPY；验收不得点击最终“确认订阅”，不得创建或修改订阅、地区映射、价格快照、目标价和通知。

## 7. 文档自检

- 本规格没有 TBD、TODO 或未决数据来源。
- 直接搜索、关系发现与最终确认的职责互不混淆。
- 唯一自动、多个候选、部分结果和完全失败的行为均已明确。
- 完整版关系根 ID 与升级包价格 ID 的边界一致，未把 `upgrade: 1` 当作商品身份。
- 范围仅覆盖日区升级包一层关系，适合单独实施和验收。
