# 仪表盘与订阅详情设计规格

## 1. 文档控制

| 项目 | 内容 |
| --- | --- |
| 状态 | 已确认，待实施 |
| 日期 | 2026-07-17 |
| 范围 | 管理员仪表盘、单个订阅详情、手动刷新入口与订阅配置编辑 |

## 2. 目标与非目标

本阶段让已登录管理员在首页总览全部订阅的五区最新价格、来源、历史最低价与采集时间，并可点击任一商品进入单页详情，直接管理订阅状态、已确认的监控地区与目标价。

本阶段不新增第三方价格抓取、不创建未确认地区商品、不修改价格采集规则，也不实现 CSV 导出、独立全局历史页或完整设置页。左侧导航保留“价格历史”和“设置”入口位置，但只有仪表盘与添加订阅在本阶段具备完整页面行为；避免用空白页面伪装未实现功能。

## 3. 页面与导航

### 3.1 应用外壳

- 认证成功后显示左侧固定导航：仪表盘、添加订阅、价格历史、设置。
- 首次认证进入仪表盘；“添加订阅”复用现有官方订阅向导。
- 本阶段由浏览器 History API 管理 `/`、`/subscriptions/new` 和 `/subscriptions/:id`，浏览器前进/返回必须恢复相应页面，不引入额外路由依赖。
- 认证失效时立即卸载全部仪表盘和详情数据，返回登录页；旧价格、地区选择和目标价不得继续留在内存中显示。

### 3.2 仪表盘（方案 A：概览优先）

- 顶部显示正在监控商品数、可用地区价格数、最近成功采集时间、下一次日报时间，以及“添加订阅”和“立即刷新”操作。
- 每款订阅为可点击整卡，显示商品名称、启用/暂停状态、五区的原始货币当前价、人民币估算、来源标记、过期或无价格状态，以及按人民币比较的跨区历史最低价。
- 卡片不显示价格曲线、地区选择或目标价编辑，保持多商品比较时的阅读密度；点击进入该订阅的详情页。
- 空状态仅提示尚未添加订阅并提供添加入口，不能虚构价格或采集时间。

### 3.3 订阅详情（单页详情）

- 顶部提供返回仪表盘、商品名称、商品类型、订阅启用/暂停状态和手动刷新入口。
- 价格区以五区卡片展示当前价、人民币估算、来源、采集时间、过期状态和地区历史最低价；无快照时明确显示“等待首笔价格”。
- 趋势区使用 `/api/history` 的不可变快照，支持全部已监控地区或单一区域筛选；折线按人民币估算绘制，缺少人民币金额的快照不参与跨区折线比较，但保留本币历史提示。
- 管理区允许直接保存三类独立配置：订阅启用状态、现有已确认地区商品的监控范围、全局人民币目标价及单区本币目标价。
- 取消勾选地区只更新订阅关联，不删除地区商品、历史快照或目标价审计；详情页不得让管理员输入地区商品 ID 或跨区复用价格 ID。
- 若管理员希望新增尚未确认映射的地区，页面必须引导至“添加订阅”的官方确认流程，而不是绕过商品身份与价格 ID 校验直接写入。

## 4. 数据与接口边界

### 4.1 读取模型

- 扩展 `GET /api/dashboard` 的概览响应，补充最近成功采集时间、下一次日报时间和可安全展示的价格状态；仍只返回管理员自己的业务数据。
- 新增 `GET /api/subscriptions/:id`，只允许管理员会话读取。该端点返回一个订阅详情所需的稳定 DTO：游戏身份、启用状态、已确认地区商品、当前/最低快照、全局目标价及单区目标价。不存在的订阅返回 `404`，不回显 SQL、会话或来源原始响应。
- 保留 `GET /api/history?subscriptionId=&region=` 作为趋势数据来源；浏览器只按订阅 ID 和受控地区代码请求，不请求任天堂、汇率或第三方站点。

### 4.2 写入模型

- `POST /api/refresh` 只排队并返回请求时间与冷却截止时间；页面显示“已排队，等待采集任务执行”，不得声称已经更新价格。
- 详情页复用 `PATCH /api/subscriptions/:id` 分别保存启用状态、非空去重地区商品 ID 列表和完整目标价配置。每次成功后重新读取详情与仪表盘，避免本地缓存覆盖并发或服务端校验结果。
- 对 `401` 统一执行安全登出；`404` 返回仪表盘并提示订阅已不存在；`422` 保留管理员尚未保存的表单输入并展示服务端脱敏中文摘要；`429` 显示刷新冷却截止时间。

## 5. 展示与可靠性规则

- 始终同时显示本币价格与可用的人民币估算；不得用零、旧币种或换算失败金额填充空值。
- 官方、未来获准的第三方、过期和无价格必须以文字状态区分。当前未获准第三方来源不会在页面制造第三方价格。
- 所有价格格式由货币决定最小单位：JPY 无小数位，其余首版地区显示两位小数；人民币以元显示，存储和请求仍使用最小单位。
- 详情趋势与仪表盘最低价必须直接使用 Worker 返回的快照，不在浏览器重新推断官方性、汇率或最低价。
- 窄屏下导航改为顶部横向入口，五区卡从五列重排为两列或单列；关键状态和操作不得依赖悬停。

## 6. 验收与测试

- Worker 测试：概览补充统计、受保护订阅详情读取、404、详情 DTO 不含敏感字段，以及既有订阅更新输入校验回归。
- 前端纯函数测试：路由解析/返回、价格与人民币格式化、趋势数据按地区筛选、无快照与过期状态转换。
- 页面测试：认证后默认进入仪表盘、卡片进入详情、返回按钮、手动刷新排队与冷却、启用/暂停、地区范围与目标价保存、401 清理状态、422 保留编辑输入。
- 浏览器验收：桌面与窄屏分别验证概览卡的五区信息可读、详情趋势与编辑区可操作、来源与过期文字可见；所有测试使用本地 D1 与模拟响应，不写入真实 Telegram 凭据或访问第三方价格站。

## 7. 影响范围

- 前端：认证成功后的根组件、同源 API 客户端、仪表盘页面、订阅详情页、共享导航、暖色主题样式与测试。
- Worker：仪表盘读取服务、订阅详情读取路由/DTO、路由注册与测试；现有订阅写入和历史读取接口保持兼容。
- 文档：需求追踪、API 设计、系统架构和质量验收在实施完成时更新为实际接口与验收结果。
