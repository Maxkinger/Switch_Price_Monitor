# 初始化与登录入口设计

**状态：**已确认，待实施  
**日期：**2026-07-17

## 1. 目标与范围

为当前“添加价格订阅”页面补齐单管理员的首次初始化、登录和恢复密码入口，使未认证访问不再只能得到“请先登录”的提示。完成首次初始化或成功登录后，管理员直接进入添加订阅向导；本设计不包含仪表盘、设置页、修改密码页或多账户功能。

## 2. 选定方案

采用单页认证状态机，不引入前端路由库或 URL 哈希路由。根组件启动时读取 `GET /api/auth/status`：未初始化显示首次设置页，已初始化显示登录页；认证成功后由同一根组件渲染现有订阅向导。

选择此方案的原因是当前站点为个人单管理员工具，已有认证接口与订阅向导均由同一 SPA 承载。状态机能复用暖色主题、减少路由与会话同步复杂度，并在后续新增仪表盘时再演进为独立页面结构。

## 3. 页面与交互

### 3.1 首次设置页

首次设置页包含：

- 管理员密码和确认密码，前端仅校验两次输入相同；密码长度与业务规则仍由服务端强制为至少 16 个字符。
- 美区、日区、墨西哥区、巴西区、香港区的启用地区多选，至少选择一个。
- 默认搜索区下拉框，只显示当前已勾选的地区；取消已作为默认项的地区时，前端自动将默认搜索区改为剩余的第一项。
- “完成初始化”按钮。初始化接口只创建认证材料，不会返回会话 Cookie；前端在恢复码确认后才以本次表单内存中的密码调用登录接口创建会话。

初始化成功后替换为一次性恢复码确认页：完整恢复码和本次密码只在内存中存在，页面不写入 URL、localStorage、sessionStorage、日志或后续 API 请求。管理员点击“我已保存，进入订阅”后，前端立即清除恢复码并以仍在内存中的密码调用登录接口；登录成功直接进入添加订阅页，登录失败则清除密码并转到登录页显示安全摘要。

### 3.2 登录页

登录页只接收管理员密码，调用 `POST /api/auth/login`。会话 Cookie 由 Worker 以 `HttpOnly`、`Secure`、`SameSite=Lax` 写入，浏览器脚本不读取、保存或拼接令牌。成功后直接进入添加订阅向导；登录页提供“使用恢复码重设密码”入口。

### 3.3 恢复密码页

恢复密码页接收恢复码、新密码和确认密码，调用 `POST /api/auth/recover`。成功后显示“密码已重设，请重新登录”，返回登录页而不自动登录；原因是服务端会撤销所有既有会话且恢复接口不应签发新会话。恢复码无效、已使用或参数不符合规则时，仅显示 Worker 返回的中文安全摘要。

## 4. 状态、接口与安全边界

根组件维护以下受控 UI 状态：`loading`、`setup`、`recovery-code`、`login`、`recover`、`authenticated`。认证 API 客户端只调用同源接口，并统一使用 `credentials: "same-origin"`：

| 接口 | 页面用途 | 成功结果 |
| --- | --- | --- |
| `GET /api/auth/status` | 决定初始显示设置页还是登录页 | `{ initialized: boolean }` |
| `POST /api/auth/initialize` | 保存管理员密码、地区和默认搜索区 | 一次性恢复码；确认保存后由内存密码触发登录 |
| `POST /api/auth/login` | 创建 Worker 管理的安全会话 Cookie | 进入订阅向导 |
| `POST /api/auth/recover` | 用一次性恢复码替换密码并撤销会话 | 返回登录页 |
| `POST /api/auth/logout` | 后续页面退出入口使用；本阶段只保留客户端能力，不新增导航 | 清除 Cookie |

任一受保护 API 返回 `401` 时，认证壳层将当前状态切换回登录页，并移除内存中的订阅向导状态、错误信息和恢复码。该处理不尝试读取 Cookie，也不把未授权响应中的商品数据留在界面上。

提交期间按钮必须禁用并显示进行中状态，防止重复初始化、登录和恢复。`409` 初始化冲突会重新读取认证状态并跳转登录页；`429` 锁定与其他 `4xx/5xx` 错误显示服务端安全摘要，不显示堆栈、请求正文、密码或恢复码。

## 5. 视觉与可访问性

认证页复用现有暖色卡片背景、中文字体、按钮和输入框样式，避免在向导前切换陌生视觉体系。每个输入具有可见标签；密码使用 `type="password"`；地区选择使用原生可访问复选框；错误和成功提示使用 `role="status"` 或 `role="alert"`。窄屏上表单改为单列，按钮保持足够触控尺寸。

## 6. 验收与测试

- 纯状态测试覆盖：默认搜索区随地区选择约束、恢复码和初始化密码只存在于一次性内存状态、恢复码确认后的自动登录、认证成功与 `401` 时的页面切换。
- API 客户端测试固定所有请求为同源 JSON 与 `credentials: "same-origin"`，并验证不将密码或恢复码写入持久化浏览器存储。
- Worker 集成测试复用既有认证路由，覆盖初始化、登录、恢复、一次性恢复码与登录锁定的安全约束。
- 浏览器级验证覆盖未初始化设置、恢复码确认、登录后进入添加订阅、恢复后回到登录，以及未认证商品搜索安全提示。
- 不在测试夹具、日志、截图文件或提交中保存真实管理员密码、真实恢复码、会话 Cookie、Telegram 凭据或其他秘密。

## 7. 边界外事项

- 认证后仪表盘、导航、主题切换、设置页、修改密码页和退出按钮的可见入口留给后续管理员 UI 工作。
- 不改变认证服务的密码哈希、锁定、会话时长、恢复码一次性和撤销逻辑。
- 不实现多用户、邮箱找回、第三方登录、验证码或密码管理器之外的凭据同步。
