# 设置驱动的地区补全设计规格

**状态：** 已确认，待实施  
**日期：** 2026-07-17

## 1. 目标

添加订阅时，美区等默认搜索区只作为官方候选的搜索入口。系统必须以服务端保存的 `enabledRegions` 作为后续地区范围，主动核验其他启用地区的任天堂官方商品；不得因为管理员提前点击“确认订阅”而静默创建只含默认区的订阅。

已有订阅也必须支持从详情页主动补全尚未映射的已启用地区，且不能删除既有地区、价格历史、目标价、启用状态或订阅身份。

## 2. 发现与确认规则

1. `POST /api/products/resolve-regions` 只接受已在默认搜索区选定的官方候选，不再接受浏览器提交的地区数组。Worker 从 `settings.enabledRegions` 读取地区范围并排除候选自身所在地区。
2. 页面选中默认区候选后自动请求跨区解析。标题、商品类型和双方可获得的发行商均一致时，返回的官方候选自动写入确认集合，并标记 `automatic`。
3. 服务不能唯一证明身份时，页面保留官方候选人工选择或本区任天堂官方链接核验入口；浏览器不能自行创建标题、价格或地区商品映射。
4. 每个启用地区必须在提交前处于“已确认”或“明确跳过”。跳过是管理员明确操作，记录在本次确认请求中，但不创建地区商品、快照或监控关联。
5. 默认区候选本身视为已确认；不存在于当前 `enabledRegions` 的既有地区不被本次操作删除。

## 3. 新建订阅接口与页面

`POST /api/products/confirm-subscriptions` 的每个商品输入新增 `skippedRegionCodes`。服务重新读取设置、重新验证全部已确认官方链接，并拒绝任一启用地区既未确认也未明确跳过的输入。所有商品、地区商品、订阅和关联仍由单个 D1 批次原子写入。

订阅向导不再维护硬编码的五区数组作为业务范围。地区标签仍可由受控展示常量生成，但解析、自动确认、提交校验和候选面板均以服务器返回的启用地区为准。自动匹配成功后可见为“已自动采用”；管理员只需处理无法可靠匹配的地区或选择跳过。

## 4. 已有订阅补全

订阅详情新增“补全已启用地区”操作。流程包含两个受认证端点：

- `POST /api/subscriptions/:id/resolve-regions`：服务从该订阅现有、已确认地区商品中读取官方身份锚点，再针对设置中尚未映射的启用地区返回相同的自动、人工选择或官方链接核验结果。不存在的订阅返回 `404`。
- `POST /api/subscriptions/:id/complete-regions`：接收本次新增的已确认候选及明确跳过地区。服务重新验证候选和订阅游戏身份，并在单个 D1 批次中只新增缺失 `regional_products` 与 `subscription_regions`。任何一个候选无法验证或地区覆盖不完整时，整次操作不写入。

补全操作不使用浏览器给出的游戏 ID、商品 URL 以外的身份声明作为事实来源；订阅 ID、游戏归属和当前地区范围均由 Worker/D1 读取。再次补全是幂等的：已有地区不重复插入，也不会替换其历史价格。

## 5. 安全与失败边界

- 浏览器始终只能调用同源受认证 API；任天堂搜索、页面解析、价格 ID 验证和 D1 写入保持在 Worker 内。
- 自动匹配绝不以数组位置、价格、语言或模糊标题单独作为身份依据。未能可靠确认时必须停在人工选择、官方链接或显式跳过。
- 设置修改只影响后续新建或主动补全操作；不得后台修改任何既有订阅。
- 401 清空向导或详情页的瞬时候选；422 保留管理员的人工链接和跳过选择；网络、D1 与官方页面细节只返回安全中文摘要。

## 6. 验收

- 生产设置为五区、默认区为 US 时，新建商品自动得到 US/JP/MX/BR/HK 的解析任务；自动匹配的地区无需再次点击“采用”。
- 任何未确认也未跳过的启用地区均使新建确认按钮不可用，且 Worker 即使收到被篡改请求也返回 `422`。
- 设置缩小为 US/JP 时，新流程只要求这两区；浏览器伪造 HK 等地区不会扩展 Worker 的解析范围。
- 已有 US 订阅补全 JP 时，订阅 ID、US 历史和目标价保持不变，只增加 JP 映射；任一新增地区失败时不产生部分写入。
- 新增和补全流程均有自动化 API/服务/状态测试，并以官方可核验的地区商品进行生产受控验收。
