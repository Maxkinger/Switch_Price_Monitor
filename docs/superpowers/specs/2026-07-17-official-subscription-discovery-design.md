# 官方订阅发现与批量确认设计规格

**状态：已确认（草图暂定）**  
**日期：2026-07-17**  
**适用范围：个人单管理员的“添加订阅”流程**

## 1. 目标

让管理员在设置的默认搜索区按名称发现任天堂官方商品，可一次选择多个不同商品；每个商品独立完成其他已启用地区的官方匹配、来源预览和订阅创建。

## 2. 数据来源与安全边界

- 商品发现、默认区搜索、跨区匹配和手动链接验证只读取任天堂官方商店或任天堂公开商品数据。第三方站点不参与候选发现或匹配。
- 价格采集仍遵循“官方优先、第三方回退”规则；第三方仅在官方价格读取失败后按设置顺序用于价格展示和日报。
- 所有任天堂请求由 Worker 在管理员会话验证后发起。浏览器不直接读取商店页面，不携带 Cookie、账号、购买信息或任天堂响应原文。
- 每个地区商品的官方价格 ID 由该地区官方链接或公开数据取得并验证；管理员不能手填，且绝不跨区复用。

## 3. 用户流程

```text
输入名称
  → 默认区官方搜索
  → 多选一个或多个候选商品
  → 每个商品分别匹配其他已启用地区
  → 自动匹配 / 选择官方候选 / 粘贴当地官方链接
  → 每个商品逐区预览官方或第三方来源
  → 批量确认并创建订阅
```

1. 默认搜索区只取 `settings.defaultSearchRegion`，不接受浏览器临时覆盖。
2. 默认区官方搜索没有结果、官方搜索适配器尚未通过准入，或候选商品不正确时，管理员可以粘贴该区任天堂官方商品链接。
3. 可同时选择多个不同商品；每个被选商品分别进入跨区确认，不能把同一地区的候选交叉分配给不同商品。
4. 对每个其他已启用地区，系统先尝试官方自动匹配。自动匹配无结果或置信度不足时，管理员可以查看当地官方候选，或粘贴当地任天堂官方商品链接。香港区与其他地区采用相同规则。
5. 来源预览复用现有逐区状态：官方价格可用、官方 ID 未确认且将使用已启用第三方回退，或无可用来源而不监控该区。
6. 最终确认前不写入 `games`、`regional_products` 或 `subscriptions`。最终确认以单个 D1 原子批次写入所选游戏、地区商品和订阅关联；某一商品无法验证时整批返回可读校验错误，不产生半成品记录。
7. 如果逻辑游戏已经订阅，返回既有订阅而不覆盖其地区配置；前端跳转至既有订阅编辑页。

## 4. 服务与接口设计

### 4.1 官方搜索与链接确认服务

地区适配器只在本区请求形式、字段与条款已经验证后才可返回候选。候选 DTO 至少包括：地区、官方链接、标题、发行商（可空）、受控商品类型、货币、官方封面链接（可空）、当前价（可空）和常规价（可空）。

- `POST /api/products/search`：输入 `{ query }`；服务端读取默认搜索区并返回该区官方候选，或返回官方搜索不可用状态。
- `POST /api/products/resolve-regions`：输入一组已选默认区候选和待确认地区；逐商品返回其他地区的自动匹配、当地官方候选入口或官方链接入口。
- `POST /api/products/preview-sources`：保留现有只读接口，输入每个最终地区候选并返回逐区来源预览。
- `POST /api/products/confirm-subscriptions`：输入经确认的多个商品及其地区商品；服务端重新校验每项身份、地区、货币和官方链接后执行原子写入，并分别返回新建或既有订阅。

所有接口要求管理员会话。匿名请求返回 `401`；空查询、未知地区、重复地区候选、无效链接、未知商品类型或商品身份冲突返回 `422`；不回显外部 HTML、JSON、堆栈或数据库错误。

### 4.2 持久化规则

- `games` 以受控规范化身份去重，保存确认后的标题、发行商、类型和可选封面链接。
- `regional_products` 按 `(game_id, region_code)` 唯一，保存该区货币、官方链接、可空官方价格 ID 和匹配来源（`automatic`、`manual_selection`、`manual_link`）。
- 一个游戏只能有一个订阅；批量操作中的每个游戏独立判定为新建或既有，既有订阅的地区范围不被隐式改写。

## 5. 候选卡与草图确认布局

- 桌面端官方候选区一行固定显示三张紧凑卡片；窄屏降为两列或一列。
- 点击整张候选卡切换选择状态；不显示“已选择”或“点击选择”文字。选中卡片使用暖色 `3px` 边框和浅色背景；可多选。
- 每张卡片使用官方封面缩略图。封面 URL 缺失或加载失败时显示无敏感数据的占位封面，标题、发行商、类型和价格仍可选择和验证。
- 以 `54×72px` 封面为垂直基准：名称距封面上缘 `2px`；商品类型在中间垂直位置且左对齐；发行商距封面下缘 `2px`。三项从同一左侧起点开始。
- 发行商和价格处在同一条封面底部水平线上：发行商在左，价格在右。
- 若官方候选有常规价和促销价且促销价低于常规价，价格区显示划线常规价、突出促销价和折扣百分比；否则只显示当前价。候选没有可验证价格时显示“价格待确认”，不得推测金额或折扣。

## 6. 失败处理与验收

- 默认区官方搜索只返回任天堂官方候选；第三方候选永不混入搜索结果。
- 未通过地区搜索准入时，页面明确显示官方搜索不可用，并提供当地官方链接验证入口。
- 多选两个商品时，跨区候选和来源预览必须按商品独立返回；同一区每个商品最多确认一个地区候选。
- 搜索、跨区匹配和来源预览前后验证 D1 记录数不变；最终批量确认才创建记录。
- 默认区手动官方链接、香港区手动官方链接、自动匹配、来源预览、批量新建和包含既有订阅的批量确认均有离线服务或路由测试。
- 所有新旧接口维持管理员会话限制，真实任天堂请求由可注入离线桩替代测试，不在测试中访问生产站点。

## 7. 范围外事项

- 本规格不承诺尚未通过 ADR-002 准入的地区具有官方自动搜索或官方价格接口；它们必须清楚地回退到官方链接确认或价格来源预告。
- 本规格不实现真实价格采集执行器、Telegram 部署配置、公开多用户订阅或支付功能。
