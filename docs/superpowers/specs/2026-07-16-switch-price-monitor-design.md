# Switch Price Monitor 解决方案设计规格

**状态：待用户最终审阅**  
**日期：2026-07-16**  
**适用范围：首版个人单管理员价格监控站**

## 1. 目标

为单一管理员提供一个简体中文的 Switch / Switch 2 价格监控站：可订阅多款商品和多个 eShop 地区，保存价格历史与人民币估算，并通过 Telegram 发送日报、官方确认的降价提醒以及采集异常/恢复通知。

## 2. 已批准方案

- 部署：Cloudflare Workers Static Assets 托管 React 前端和 API，D1 保存数据，Cron 执行采集和调度，Secrets 保存敏感凭据。
- 首页：左侧导航仪表盘；默认主题为温暖游戏库，可切换沉稳深色和清爽工具。
- 地区：首次初始化时由管理员从美、日、墨西哥、巴西、香港五区选择至少一个初始监控地区，并从已选地区中指定默认搜索区。设置可开启更多地区；修改默认搜索区不改变既有订阅。
- 数据：官方任天堂价格优先；失败后按管理员配置的第三方优先级回退。默认候选为 eShop-Prices、NT Deals，Deku Deals 与 Green Pipe 可选；来源的生产准入受 ADR-002 约束。
- 通知：每 6 小时采集；日报默认每天 `Asia/Shanghai` 09:00；官方降价即时提醒；同轮事件合并，长日报自动分页。日报与降价汇总中每款商品附带历史最低价。

## 3. 核心业务规则

1. 添加商品时，管理员在默认区确认候选商品；系统自动匹配其他地区，低置信度时允许候选选择或粘贴官方链接。
2. 若重复添加已订阅商品，打开既有订阅编辑页，不创建重复记录。
3. 当前价格优先显示官方数据；第三方数据必须标明站点。所有来源失败时保留最后成功价格并标为可能过期。
4. 每日统一使用一次读取的中间汇率展示人民币估算；失败时沿用最近成功汇率并标明过期。美区同时显示标价和按所选州估算的含税价，默认州为 Oregon。
5. 仅官方价格低于上一条官方成功价格时，触发即时降价提醒。第三方价格只用于页面和日报。
6. 支持全局 CNY 目标价和单区当地货币目标价；单区优先。只有首次跌破目标价提醒，回升后再次跌破才再次提醒。
7. 日报和即时降价汇总对每款商品展示历史最低价：全监控区最低价所在地区、当地货币、按记录时汇率计算的人民币估算及日期；同时提供各区各自的历史最低当地货币价格。
8. 取消订阅采用软停用并保留历史。价格历史默认永久保存，也可保留一年或两年；采集日志固定保留 90 天。

## 4. 组件与数据流

```text
React UI → Worker API → [官方商店 → 第三方回退 / 汇率服务 / Telegram]
                       ↓
                      D1
```

- UI 只访问 Worker API；不会直接访问外部商店、第三方站点、汇率服务或 Telegram。
- Worker 按地区商品采集，执行商品身份校验、价格快照写入、汇率关联和健康状态计算。
- D1 保存认证、设置、商品/地区映射、订阅、价格快照、汇率、采集日志、健康状态和通知事件。
- 采集连续失败 3 次后发送一次异常通知；下次成功时发送恢复通知。

完整实体定义见 [数据模型](../../architecture/data-model.md)，接口边界见 [API 设计](../../architecture/api-design.md)。

## 5. 安全与可靠性

- 首次访问进行一次性管理员初始化：设置密码、选择初始地区和默认搜索区；密码使用安全哈希、会话使用安全 Cookie、登录限流，恢复码仅显示一次并以不可逆校验值保存。
- 外部请求有超时设置；临时网络错误最多重试一次，再尝试下一来源。
- 商品标题、发行商和类型校验不匹配时不得写入价格。
- Telegram 凭据优先从 Cloudflare Secrets 读取；设置页凭据不得回显、写入导出或普通日志。
- CSV 只导出订阅配置、价格历史或采集日志，并排除所有认证与密钥字段。

## 6. 验收标准

- 五个默认地区至少各有一个真实商品通过采集验证。
- 验证官方优先、第三方回退、过期价格/汇率标记、官方降价通知和第三方不即时通知。
- 验证目标价去重、日报中每款商品的全区/单区历史最低价、日报分页、三连失败与恢复通知、初始化/登录/恢复码、导出字段安全。
- 生产发布前先在测试环境验证 D1 迁移、Telegram、定时任务，发布后执行一次受控手动刷新。
