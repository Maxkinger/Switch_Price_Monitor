# 订阅硬删除与全局请求加载设计规格

**状态：** 已部署；公开可用性与两条生产删除路径均已验证
**日期：** 2026-07-18

## 1. 目标

管理员可在仪表盘多选订阅后永久删除，也可在单个订阅详情页永久删除；删除成功后详情页回到仪表盘。任何站内 API 请求执行期间，已认证页面均显示统一的全局加载动画，请求结束（成功或失败）立即撤销。

## 2. 硬删除边界

- 删除不是现有的软停用：操作不可恢复，删除确认弹窗必须明确说明“订阅、地区映射、价格历史、采集日志和通知事件将永久删除”。
- 仪表盘卡片提供独立的多选控件，点击卡片正文仍进入详情；选中一项或多项后显示“删除已选（N）”。
- 详情页提供“删除订阅”危险操作。操作成功后使用现有路由机制进入仪表盘，并重新读取概览。
- 两个入口共用同一可访问的确认弹窗：默认焦点和默认按钮为“取消”，明确点击“永久删除”才发送请求；请求进行时禁用关闭、取消和确认，避免重复删除。
- `DELETE /api/subscriptions` 接受去重且非空的 `subscriptionIds` 数组。服务端先验证所有订阅都存在，再在一个 D1 原子批次中删除全部目标订阅；任一 ID 无效则返回 `404` 且不删除任何记录。
- 每个订阅的游戏在本系统中保持一对一；服务端按依赖顺序删除目标价、关系、通知事件、地区健康状态、价格快照、采集日志、地区商品、订阅和游戏。全局汇率、设置、管理员、会话与其他订阅绝不受影响。

## 3. 全局加载模型

- 新增前端 `ApiRequestTracker`，以非负的进行中请求计数为唯一状态；`begin()` 返回一次性结束函数，必须在 `finally` 调用。
- 认证、设置、仪表盘/详情、商品搜索与订阅确认等全部同源 API 客户端共用该跟踪器。页面组件不自行猜测网络状态，也不以某个局部按钮状态替代全局状态。
- 根应用在已认证壳层渲染全局加载遮罩：含动画、`role="status"` 与“正在同步数据…”文本；遮罩阻止重复点击，但不显示请求路径、请求体、Cookie 或服务端错误。
- 并发请求以计数为准：第一个请求开始时显示，最后一个请求在 `finally` 结束后才隐藏；失败、401、422 和 429 同样会清理计数，不能留下永久遮罩。
- 认证前沿用现有各表单的局部文案，避免把设置管理员密码等敏感流程与已认证业务页面混在同一个遮罩中。

## 4. 受控错误与导航

- 删除请求的 `401` 沿用现有安全登出；`404` 不泄露任何其他订阅信息，仪表盘重新读取并提示“订阅已不存在”；`422` 保留确认弹窗并显示服务端脱敏摘要。
- 删除成功后仪表盘清空本次多选状态并通过 Worker 返回的新读取模型展示结果；不在浏览器手工裁剪价格、统计或历史数据。
- 详情页删除成功后先清除本页局部详情/历史草稿，再导航仪表盘，避免浏览器返回时短暂显示已经删除的价格。

## 5. 测试与验收

- Worker：批量删除的认证、空/重复输入、任一不存在时零删除、多个有效订阅的全依赖清理、其他游戏和全局数据不受影响、原子失败回归。
- 前端状态/页面：仪表盘独立多选不触发导航；删除确认成功后刷新概览并清空选择；详情删除成功后返回仪表盘；取消和失败不改变页面数据。
- 请求跟踪：单请求、并发请求、抛出错误与 401 均在结束后恢复到零；遮罩在计数大于零期间可见且无敏感文本。
- 浏览器：多选与详情两条删除入口均需人工确认后才发生永久写入；验证动画在搜索、保存、刷新和删除期间出现，请求结束后消失。

## 6. 非目标

- 不新增恢复站、回收站、定时延迟删除或管理员审计导出。
- 不删除全局汇率、设置、管理员、会话、恢复码或其他订阅的业务数据。
- 不把加载动画用于任天堂页面、Telegram 或任何第三方页面；它只表现本站同源 API 的请求状态。
