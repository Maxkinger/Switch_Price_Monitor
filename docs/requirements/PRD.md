## 2. 产品目标与范围

本产品用于监控 Nintendo eShop 中多款 Switch / Switch 2 商品在多个地区的价格，保存价格历史、换算人民币，并通过 Telegram 推送每日价格日报和官方确认的降价提醒。

首版支持完整游戏、Nintendo Switch 2 升级包，以及由管理员手动开启的 DLC、季票、豪华版、组合包和其他附加内容。

不在首版范围内：多用户注册、公开订阅、支付、移动 App。

## 3. 已确认功能需求

### FR-001 订阅与区域管理

- 支持订阅多款商品；每款商品可选择不同监控地区。
- 首次初始化时，管理员从美国、日本、墨西哥、巴西、香港五区中选择要启用的初始监控地区；至少选择一个地区。设置页可在之后启用其他任天堂 eShop 地区。
- 首次初始化时，管理员从已选初始地区中指定默认搜索区；后续可修改。修改只影响后续新增商品，不改变既有订阅。
- 添加流程：输入名称 → 搜索默认区候选商品 → 管理员确认 → 按设置中已启用地区自动匹配其他地区 → 处理无法可靠匹配的地区或明确跳过 → 创建订阅。默认搜索区只决定搜索入口，不会限制后续监控地区。
- 默认区搜索、跨区候选匹配和手动链接验证只使用任天堂官方商店或任天堂公开商品数据；第三方站点只可作为官方价格失败后的价格回退。
- 默认区候选支持一次选择多个不同商品，并由每个商品独立完成跨区确认、来源预览和订阅创建。
- 默认区官方搜索无结果、尚未通过地区适配器准入或候选不正确时，管理员可粘贴默认区任天堂官方商品链接。
- 搜索和跨区匹配阶段应从官方商品链接或公开商品数据自动取得每个地区各自的官方价格 ID；不得要求管理员手动填写价格 ID。
- 创建订阅前必须逐区告知官方价格状态：官方价格可用、官方价格 ID 未确认（并列出将使用的第三方站点）或无可用价格来源。管理员确认预览后才可创建订阅。
- 身份完全一致的跨区官方候选自动确认；自动匹配不可靠时，管理员可从当地候选列表选择、直接粘贴该区任天堂官方商品链接，或明确跳过该区。每个启用地区在创建前必须被确认或跳过，不能静默创建仅默认区订阅。
- 已验证的跨语言官方组合商品可自动确认，但必须同时满足：同为 `bundle`、同发行商、相同拉丁主标题、相同受控版本标记且本区候选唯一。美食家版当前准入英文 `Gourmet Edition` 与日文 `真の食通エディション`；当英文日区查询无同类型结果时，仅可从唯一官方同系列结果提取一个日文别名重试一次。日区最终复核以候选官方日文标题查询并继续绑定精确 URL 与价格 ID；原版、Switch 2 Edition、升级包、DLC 和季票不得因此规则自动匹配。
- 港区名称检索只请求可由 Worker 稳定访问的香港普通官网一次，不请求会拒绝 Worker 的 Magento 商城搜索。首轮没有同类型候选时，仅允许对已准入的 `Gourmet Edition` 或 Switch 2 Upgrade Pack 后缀生成一次基础标题搜索；结果必须为 1–5 个港区官方 `titles/{ID}` 本体。
- 港区基础本体详情只展开一层任天堂官方 `includedBundleItems`、`dlcItems.items` 和 `upgradeInfo` 关系，单根最多 50 个去重关联。组合商品、DLC 和升级包均须重新读取自己的 `bundles/{ID}` 或 `aocs/{ID}` 详情并验证地区、URL、类型和发行商；任一根或关联详情不完整时回退人工官方链接，不使用部分结果自动确认。
- 非日区 `automatic` 候选在最终写入前必须重新执行同一官方发现和唯一性规则，并仍得到完全相同的官方 URL；失效时整批零写入。人工选择不依赖该自动唯一性重验，日区继续使用其软件搜索与价格 API 专用复核。
- 每个地区商品记录匹配来源：自动匹配、手动选择或手动链接。
- 候选卡展示官方封面、标题、类型、发行商和官方可验证价格；存在促销时划线显示原价、显示折扣价及折扣力度。桌面端一行显示三个候选，点击整张卡片支持多选，选中状态仅以边框和背景表示。
- 取消订阅时仅停止后续采集和通知，保留既有价格历史；管理员可重新启用订阅。
- 永久删除是独立于取消订阅的不可恢复操作：仅管理员确认后才可批量删除指定订阅，并同时清理其地区映射、目标价、价格历史、采集日志、健康状态、通知事件、地区商品和游戏；不得删除其他订阅、全局汇率、设置或认证资料。
- 新建订阅时若识别为已订阅商品，不创建重复记录；应打开既有订阅编辑页。详情页提供“补全已启用地区”，以现有已确认地区商品为身份锚点并只原子新增缺失映射，不删除既有历史或配置。
- 日区升级包未作为独立名称搜索结果返回时，系统可从唯一、同系列、同发行商且由官方标记可升级的 Switch 2 Edition 下载版详情读取一层“アップグレードパス”关系。升级包必须使用关系链接自己的官方 URL 与价格 ID，并经 JP、JPY、在售状态和同 ID 价格响应复核；唯一完整候选自动加入，多个完整候选由管理员选择，部分结果、排队页或结构异常不得自动加入。最终确认须重新建立同一关系证明，不能信任浏览器提交的 automatic 状态。

### FR-002 数据来源与展示

- 任天堂官方商店为首选数据源。
- 官方读取失败时，按设置的优先级读取已启用第三方站点。
- 默认候选并排序的第三方站点：eShop-Prices、NT Deals；Deku Deals、Green Pipe 作为可选站点，由管理员启用、停用和排序。各来源须先通过 [ADR-002](../decisions/ADR-002-price-provider-validation.md) 的分区准入验证，才可在生产采集默认启用。
- 有官方价格时必须展示官方价格；无官方价格时可展示第三方价格，并在价格后清楚标记来源。
- 已确认的官方价格 ID 只能用于同一地区；官方接口响应的地区、货币或价格 ID 不匹配时，必须拒绝该官方结果并按设置回退。后续补齐官方价格 ID 只影响新快照，既有第三方历史保留原来源标记。
- 所有来源失败时，保留最近一次成功价格，显示“数据可能过期”与最后成功获取时间。
- 始终显示地区原始货币价格；人民币价格采用每日一次的中间汇率换算，并在当天统一使用同一批汇率。
- 系统使用内置的可靠汇率服务自动读取每日中间汇率，并展示汇率来源及更新时间。
- 当日汇率读取失败时，沿用最近一次成功汇率，并标记“汇率可能过期”；日报仍正常发送。
- 美区同时展示未税标价及含税估算价；初始税务州为俄勒冈州（无州销售税），可在设置中选择其他州。

### FR-003 价格采集、历史与手动刷新

- 自动采集频率为每 6 小时一次。
- 管理员可手动刷新；临时开发验证期间取消 15 分钟冷却，每次已认证点击都立即执行一次与自动采集相同的官方采集轮次并返回本轮统计，同时仅记录最近刷新时间。六小时自动采集保持独立运行，不消费手动刷新请求；恢复冷却须另行确认并重新启用服务端限流。
- 默认永久保留历史价格；设置可切换为仅保留最近一年或两年，并自动清理超期数据。
- 每条历史记录须保留采集时间、原始金额/货币、人民币金额/汇率、数据来源、价格口径和采集状态。
- 采集日志独立于价格历史，固定保留 90 天后自动清理。

### FR-004 降价与目标价提醒

- 仅当官方价格低于上一条官方成功记录时，触发即时降价提醒。
- 第三方价格用于展示和日报，不单独触发即时降价提醒。
- 同一采集轮次内的多个官方降价合并成一条 Telegram 汇总消息，按商品和地区分组。
- 每个商品支持全局人民币目标价；也支持按地区设置当地货币目标价。单区目标存在时优先使用。
- 目标价命中时须在提醒中明确说明。
- 同一目标价仅在价格首次跌破时提醒；价格回升到目标价之上后再次跌破时，才允许再次提醒。

### FR-005 日报与 Telegram

- 默认时区为 `Asia/Shanghai`，默认日报时间为每天 09:00；设置页可修改时区和时间。
- 全部订阅合并为一份日报，展示每款商品的全部监控区价格。
- Telegram 日报和即时降价汇总中，每款商品均展示历史最低价信息：最低价所在地区、当地货币金额、按当时汇率计算的人民币估算价及记录日期；多地区时同时展示各区各自的历史最低当地货币价格。
- 超出 Telegram 单条消息长度时，自动拆分连续消息并标注页码。
- 配置方式同时支持部署环境变量和设置页填写；环境变量优先。
- 设置页提供 Telegram 测试消息；Token 和 Chat ID 不得返回前端或写入日志。

### FR-006 身份验证与恢复

- 不开放注册，仅单一管理员登录。
- 首次访问显示一次性初始化页面，用于设置管理员密码、选择初始监控地区并指定默认搜索区；完成后不可再次通过该页面初始化。
- 密码仅以安全哈希保存；登录接口必须限流，连续失败应临时锁定。
- 首次初始化生成恢复码，且只显示一次；恢复码可用于重设管理员密码。

### FR-007 网页信息架构

- 首页采用左侧导航布局。
- 左侧固定入口：仪表盘、添加订阅、价格历史、设置。
- 左侧导航底部显示构建时注入的发布版本号，格式为 `V x.y.z`；临时阶段以 `package.json` 为唯一版本来源，生产部署前自动递增补丁号，普通本地构建、开发和测试不得改变版本。
- 仪表盘展示最新价格、当前最低价、下次日报/最近更新时间和手动刷新入口。
- 初始视觉主题为“温暖游戏库”（C）；设置页支持切换“沉稳深色”（A）、“清爽工具”（B）和“温暖游戏库”（C），并保存管理员主题偏好。
- 首版网页和 Telegram 推送仅使用简体中文。

### FR-008 数据导出

- 管理员可导出 CSV 格式的数据，范围包括订阅配置、价格历史和采集日志。
- 导出仅对已登录管理员开放。
- 导出内容不得包含管理员密码哈希、恢复码校验值、Telegram Token、Chat ID 或其他敏感配置。

### FR-009 采集异常通知

- 同一地区商品连续 3 次未能从官方或已启用第三方来源取得价格时，发送一次 Telegram 异常提醒。
- 后续恢复取得价格时，发送一次恢复通知。
- 异常提醒应包含商品、地区、连续失败次数、最近成功获取时间和最近错误摘要。

## 4. 非功能需求

- 所有敏感值（Telegram Token、恢复相关密钥等）不得以明文写入源代码、版本控制或普通日志。
- 数据来源和获取时间可追溯；第三方数据与官方数据必须在 UI 中可区分。
- 对外部商店和第三方站点的读取须具备超时、失败记录和限流保护。
- 临时网络失败可重试一次后再切换下一优先级来源；商品名称、发行商或类型校验不匹配时不得写入价格。
- 发布前必须在测试环境验证 Telegram、D1 迁移、定时任务及五区真实商品采集，并在生产部署后进行受控手动刷新验证。
- 页面以个人桌面管理场景为主，同时保持基本窄屏可用性。
