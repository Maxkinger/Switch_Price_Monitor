## 2. 产品目标与范围

本产品用于监控 Nintendo eShop 中多款 Switch / Switch 2 商品在多个地区的价格，保存价格历史、换算人民币，并通过 Telegram 推送每日价格日报和官方确认的降价提醒。

首版支持完整游戏、Nintendo Switch 2 升级包，以及由管理员手动开启的 DLC、季票、豪华版、组合包和其他附加内容。

不在首版范围内：多用户注册、公开订阅、支付、移动 App。

## 3. 已确认功能需求

### FR-001 订阅与区域管理

- 支持订阅多款商品；每款商品可选择不同监控地区。
- 首次初始化时，管理员从美国、日本、墨西哥、巴西、香港五区中选择要启用的初始监控地区；至少选择一个地区。设置页可在之后启用其他任天堂 eShop 地区。
- 首次初始化时，管理员从已选初始地区中指定默认搜索区；后续可修改。修改只影响后续新增商品，不改变既有订阅。
- 添加流程：输入名称 → 搜索默认区候选商品 → 管理员确认 → 自动匹配其他地区 → 选择监控地区 → 创建订阅。
- 自动匹配不可靠时，管理员可从当地候选列表选择，或直接粘贴该区官方商品链接。
- 每个地区商品记录匹配来源：自动匹配、手动选择或手动链接。
- 取消订阅时仅停止后续采集和通知，保留既有价格历史；管理员可重新启用订阅。
- 新建订阅时若识别为已订阅商品，不创建重复记录；应打开既有订阅编辑页，以补充地区或调整配置。

### FR-002 数据来源与展示

- 任天堂官方商店为首选数据源。
- 官方读取失败时，按设置的优先级读取已启用第三方站点。
- 默认启用并排序的第三方站点：eShop-Prices、NTPrices；Deku Deals、Green Pipe 作为可选站点，由管理员启用、停用和排序。
- 有官方价格时必须展示官方价格；无官方价格时可展示第三方价格，并在价格后清楚标记来源。
- 所有来源失败时，保留最近一次成功价格，显示“数据可能过期”与最后成功获取时间。
- 始终显示地区原始货币价格；人民币价格采用每日一次的中间汇率换算，并在当天统一使用同一批汇率。
- 系统使用内置的可靠汇率服务自动读取每日中间汇率，并展示汇率来源及更新时间。
- 当日汇率读取失败时，沿用最近一次成功汇率，并标记“汇率可能过期”；日报仍正常发送。
- 美区同时展示未税标价及含税估算价；初始税务州为俄勒冈州（无州销售税），可在设置中选择其他州。

### FR-003 价格采集、历史与手动刷新

- 自动采集频率为每 6 小时一次。
- 管理员可手动刷新；每次刷新后冷却 15 分钟。
- 默认永久保留历史价格；设置可切换为仅保留最近一年或两年，并自动清理超期数据。
- 每条历史记录须保留采集时间、原始金额/货币、人民币金额/汇率、数据来源、价格口径和采集状态。
- 采集日志独立于价格历史，固定保留 90 天后自动清理。

### FR-004 降价与目标价提醒

- 仅当官方价格低于上一条官方成功记录时，触发即时降价提醒。
- 第三方价格用于展示和日报，不单独触发即时降价提醒。
- 同一采集轮次内的多个官方降价合并成一条 Telegram 汇总消息，按商品和地区分组。
- 每个商品支持全局人民币目标价；也支持按地区设置当地货币目标价。单区目标存在时优先使用。
- 目标价命中时须在提醒中明确说明。
- 同一目标价仅在价格首次跌破时提醒；价格回升到目标价之上后再次跌破时，才允许再次提醒。

### FR-005 日报与 Telegram

- 默认时区为 `Asia/Shanghai`，默认日报时间为每天 09:00；设置页可修改时区和时间。
- 全部订阅合并为一份日报，展示每款商品的全部监控区价格。
- Telegram 日报和即时降价汇总中，每款商品均展示历史最低价信息：最低价所在地区、当地货币金额、按当时汇率计算的人民币估算价及记录日期；多地区时同时展示各区各自的历史最低当地货币价格。
- 超出 Telegram 单条消息长度时，自动拆分连续消息并标注页码。
- 配置方式同时支持部署环境变量和设置页填写；环境变量优先。
- 设置页提供 Telegram 测试消息；Token 和 Chat ID 不得返回前端或写入日志。

### FR-006 身份验证与恢复

- 不开放注册，仅单一管理员登录。
- 首次访问显示一次性初始化页面，用于设置管理员密码、选择初始监控地区并指定默认搜索区；完成后不可再次通过该页面初始化。
- 密码仅以安全哈希保存；登录接口必须限流，连续失败应临时锁定。
- 首次初始化生成恢复码，且只显示一次；恢复码可用于重设管理员密码。

### FR-007 网页信息架构

- 首页采用左侧导航布局。
- 左侧固定入口：仪表盘、添加订阅、价格历史、设置。
- 仪表盘展示最新价格、当前最低价、下次日报/最近更新时间和手动刷新入口。
- 初始视觉主题为“温暖游戏库”（C）；设置页支持切换“沉稳深色”（A）、“清爽工具”（B）和“温暖游戏库”（C），并保存管理员主题偏好。
- 首版网页和 Telegram 推送仅使用简体中文。

### FR-008 数据导出

- 管理员可导出 CSV 格式的数据，范围包括订阅配置、价格历史和采集日志。
- 导出仅对已登录管理员开放。
- 导出内容不得包含管理员密码哈希、恢复码校验值、Telegram Token、Chat ID 或其他敏感配置。

### FR-009 采集异常通知

- 同一地区商品连续 3 次未能从官方或已启用第三方来源取得价格时，发送一次 Telegram 异常提醒。
- 后续恢复取得价格时，发送一次恢复通知。
- 异常提醒应包含商品、地区、连续失败次数、最近成功获取时间和最近错误摘要。

## 4. 非功能需求

- 所有敏感值（Telegram Token、恢复相关密钥等）不得以明文写入源代码、版本控制或普通日志。
- 数据来源和获取时间可追溯；第三方数据与官方数据必须在 UI 中可区分。
- 对外部商店和第三方站点的读取须具备超时、失败记录和限流保护。
- 临时网络失败可重试一次后再切换下一优先级来源；商品名称、发行商或类型校验不匹配时不得写入价格。
- 发布前必须在测试环境验证 Telegram、D1 迁移、定时任务及五区真实商品采集，并在生产部署后进行受控手动刷新验证。
- 页面以个人桌面管理场景为主，同时保持基本窄屏可用性。
